Vector v0.7.0 Release Notes

The Great Omelet
Jan 22nd, 2020 / 5 months ago
download
  • 7 highlights
  • 13 new features
  • 10 enhancements
  • 12 bug fixes
  • This is an outdated version. Outdated versions maybe contain bugs. It is recommended to use the latest version (0.9.2).

You can't make an omelet without cracking some eggs. We have our eyes set on version 1 and a backlog of breaking changes we wanted to merge before we reach it. Version 0.7.0 is our attempt to cluster as many breaking changes as we can into a single release.

#Highlights

Noteworthy changes in this release

  • # Breaking Changes

    • Merge existing `tcp` and `udp` sources into a single `socket` source

      We've simplified our socket based sources into a single `socket` source

    • Rename existing `tcp` sink to `socket` sink

      This renames the existing `tcp` sink to `socket`

    • Sink options have been refactored

      We've simplified and organized our sink options

    • Use comma delim server list in `kafka` sink

      This change achieve consistency with our `kafka` source and other Kafka clients

    • Rename `line` field to `message` in `splunk_hec` source

      This change ensures the `splunk_hec` source conforms to Vector's schema

    • The `splunk_hec` sink does not index fields by default

      This gives you full control over which fields are indexed

  • # New Features

    • New Prometheus Source

      Scrape prometheus metrics with Vector
